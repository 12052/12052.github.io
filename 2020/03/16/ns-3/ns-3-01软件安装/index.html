ns-3/ns-3-01软件安装

ns-3 01 软件安装

01 软件下载

ns-3的软件下载网址传送门,版本为3.28，文件大小为25M左右，有其他版本需求在左侧选择所需版本。

02 环境配置

本文主要讲述Ubuntu环境下的环境配置，可在此网站传送门选择其他系统的安装教程。

  • 该软件在Ubuntu19.04环境下安装运行没有问题，在其他旧版本下可能会有问题。
  • 最低C++\python编译器配置
  • Python用户需要绑定python驱动
  • 对于从git仓库拉取ns-3软件的用户需要安装git工具
  • ns-3界面演示需要用到qt5及相关组件，qt4也能用但是后面会转换到qt5
  • 对于ns-3-pyviz visualizer工具（该工具应该是做仿真时的必须工具，我当时就是在这个上实在编译不过就只能选择3.27）
    1 对于3.28及更早的版本，PyViz基于GTK+2、GooCanvas、GraphViz。 2 对于ubuntu18.04,python-pygoocanvas不再被支持，3.29及更新版本需要GTK+3和下列包
  • 基于MPI的仿真工具
  • bake build tool支持
  • Debug工具
  • utils/check-style.py 编程风格支持
  • Doxygen 相关 若报错convert ... not authorized source-temp/figures/lena-dual-stripe.eps,按照此文章描述的修改安全设置传送门
  • ns-3手册位于doc/tutorial, doc/manual, doc/models
  • GSL 来确保802.11b WiFi 模型正常工作 ubuntu16.04没有libgsl23 libgslcblas0这两个包。
    若未能正常工作，参考传送门
  • 读取pcap追踪包
  • 稳定数据库架构
  • 基于Xml
  • 生成python啥啥啥
  • 基于GTK的确认系统
  • 使用ns-3和模拟设备实验
  • 开放模组支持

    03 手动安装

    3.1 下载

    下载源码传送门，解压之后包含的文件包括build.py* constants.py dist.py* download.py* README util.py

    3.2 编译

    其实以上安装重要的是pyviz这个模组，用于将仿真过程可视化，所以下面build时一定要注意所执行的时候出现的PyViz是否为enable。
    运行./build.py --enable-examples --enable-tests后，等待编译将近十多分钟，测试位于ns***目录下的./test.py即可。

    04 例程

    运行./waf --run hello-simulator语句即可得到Hello Simulator,这可以是系统确保ns-3可以顺利运行。
